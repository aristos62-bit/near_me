# P2.1 — Provider Cascade: Διπλό create/dispose σε auth change

## Κατάσταση
- **Severity:** P2 (Optimization)
- **Status:** Fixed — Session 152
- **Αρχεία:** `chat_provider.dart`, `settings_screen.dart`, `main.dart`

---

## Περιγραφή

Σε κάθε auth state change (sign-in/sign-out), το `chatsProvider` κάνει dispose + create **2-3 φορές** αντί για 1. Κάθε dispose/create δημιουργεί νέο Firestore stream listener (collection snapshot), αυξάνοντας το Firestore read cost χωρίς όφελος.

---

## Root Cause Analysis

### Provider dependency graph (πριν το fix)

```
authStateProvider (StreamProvider, Firebase userChanges())
  │
  │ [ref.watch] ← chatsProvider (StreamProvider)
  │                 └── unreadBadgeProvider (Provider, kept alive by ref.listen)
  │
  │ [ref.listen] ← settings_screen (rebuild σε auth change)
  │
  └── [ref.listen] ← main.dart (P1.3: stop idle timer)
```

### Δύο ανεξάρτητες πηγές recreation

1. **Explicit**: `ref.invalidate(chatsProvider)` στο `settings_screen._signOut()` (από Session 151)
2. **Implicit**: `ref.watch(authStateProvider)` στη γραμμή 17 του `chat_provider.dart`

Κάθε source προκαλεί ξεχωριστό dispose/create cycle. Την ίδια στιγμή που το `streamChats()` ήδη έχει guards (`isSigningOut`, `null user`, `canUserCommunicate`) που προστατεύουν από PERMISSION_DENIED.

### Log evidence — sign-out (πριν)

```
20:58:12.000  ref.invalidate(chatsProvider)       ← explicit (settings_screen)
               chatsProvider disposed
               chatsProvider created (StreamProvider)
               streamChats: blocked (signing out)

20:58:12.965  userChanges() emits null            ← implicit (ref.watch στο provider)
               chatsProvider disposed
               chatsProvider created (StreamProvider)
               streamChats: blocked (null user)
```

= **2 cycles**, 2× Firestore listener effort

### Log evidence — sign-in (πριν)

```
20:58:37.950  userChanges emits uid=X             ← authStateProvider αλλάζει
               chatsProvider disposed
               chatsProvider created (StreamProvider)
               streamChats: started
```

= **1 cycle** (από Session 132 fix: `userChanges()` αντί `authStateChanges()`)

## Λύση

### Στρατηγική

Κατάργηση της implicit recreation (ref.watch) και συγκέντρωση όλης της λογικής recreation σε ένα σημείο — τον κεντρικό listener του `_NearMeAppState.build()`. Αυτό δημιουργεί **ένα source of truth** για το πότε το `chatsProvider` πρέπει να γίνει recreate.

### Κριτήρια recreation

Το `chatsProvider` πρέπει να γίνει recreate όταν:
1. **uid αλλάζει** (sign-in, sign-out, εναλλαγή λογαριασμού)
2. **emailVerified αλλάζει** (unverified→verified — επηρεάζει `canUserCommunicate`)

ΔΕΝ πρέπει να γίνει recreate όταν:
- Token refresh (userChanges emits ίδιο user)
- `reload()` χωρίς αλλαγή emailVerified
- Profile metadata changes (όνομα, φωτογραφία κλπ.)

### Αλλαγή 1 — chat_provider.dart:17

**Αφαίρεσε** `ref.watch(authStateProvider);`

```dart
final chatsProvider = StreamProvider<List<ChatCacheTableData>>((ref) {
  DebugConfig.log(DebugConfig.providerCreate, 'chatsProvider created (StreamProvider)');
  final chatRepo = ref.watch(chatRepositoryProvider);
  // ref.watch(authStateProvider); ← REMOVED
  final stream = chatRepo.streamChats();
  ref.onDispose(() => DebugConfig.log(DebugConfig.providerDispose, 'chatsProvider disposed'));
  return stream;
});
```

**Σκεπτικό:** Το `streamChats()` ελέγχει `_auth.currentUser` κάθε φορά που καλείται — δεν χρειάζεται το provider dependency. Το `FirebaseAuth.instance.currentUser` είναι synchronous, διαθέσιμο αμέσως.

### Αλλαγή 2 — settings_screen.dart:63

**Αφαίρεσε** `ref.invalidate(chatsProvider);`

```dart
    DebugConfig.log(DebugConfig.providerDispose,
        '_signOut: invalidating Firestore stream providers');
    // ref.invalidate(chatsProvider); ← REMOVED
    ref.invalidate(incomingRequestsProvider);
    ref.invalidate(outgoingRequestsProvider);
    ref.invalidate(unreadBadgeProvider);
```

**Σκεπτικό:** Ο κεντρικός listener (Αλλαγή 3) θα invalidate το `chatsProvider` όταν το `authStateProvider` εκπέμψει `null`. Το `isSigningOut` flag μπαίνει στην αρχή του `signOut()`, οπότε αν κάποιο rebuild συμβεί στο ενδιάμεσο, το νέο `streamChats()` μπλοκάρεται από τον guard. Ο παλιός Firestore listener σκοτώνεται στο `finally` όταν ο provider γίνει dispose.

**Window PERMISSION_DENIED:** Μεταξύ `_auth.signOut()` και `userChanges() null emission`, ο παλιός listener μπορεί να πάρει 1 error, που πιάνεται από `onError` handler — ίδιο με το σημερινό CYCLE #1 behavior. Αποδεκτό.

### Αλλαγή 3 — main.dart

**α) Προσθήκη import:**

```dart
import 'features/chat/providers/chat_provider.dart';
```

**β) Επέκταση authStateProvider listener (lines 351-358):**

```dart
    ref.listen(authStateProvider, (prev, next) {
      final prevUser = prev?.value;
      final nextUser = next.value;
      // P2.1: Invalidate chatsProvider when uid or emailVerified changes
      if (mounted) {
        final uidChanged = prevUser?.uid != nextUser?.uid;
        final emailVerifiedChanged =
            prevUser?.emailVerified != nextUser?.emailVerified;
        if (uidChanged || emailVerifiedChanged) {
          ref.invalidate(chatsProvider);
          if (uidChanged) {
            DebugConfig.log(DebugConfig.authFlow,
                'main: uid changed ${_maskUid(prevUser?.uid)} → ${_maskUid(nextUser?.uid)}');
          }
          if (emailVerifiedChanged) {
            DebugConfig.log(DebugConfig.authFlow,
                'main: emailVerified changed ${prevUser?.emailVerified} → ${nextUser?.emailVerified}');
          }
          DebugConfig.log(DebugConfig.providerDispose,
              'main: invalidated chatsProvider (auth change)');
        }
      }
      // P1.3: Stop idle timer on sign-out
      if (prevUser != null && nextUser == null && mounted) {
        _stopIdleTimer();
        if (_isLocked) {
          setState(() => _isLocked = false);
        }
      }
    });
```

**Σημείωση:** Το `_maskUid()` δεν υπάρχει στο main.dart — για τα logs θα χρησιμοποιήσουμε `nextUser?.uid ?? "null"` (χωρίς masking, όπως ήδη γίνεται στα υπόλοιπα logs).

## Ανάλυση Edge Cases

| # | Σενάριο | Συμπεριφορά | Ορθότητα |
|---|---|---|---|
| 1 | **App startup** — listener fires (prev=null, next=user) | `chatsProvider` may not yet exist → `ref.invalidate` = no-op. Later created fresh. | ✅ |
| 2 | **Rapid sign-in/out** (3×) | `ref.invalidate` idempotent — 1 cycle each | ✅ |
| 3 | **signOut() throws exception** | `userChanges()` δεν εκπέμπει → listener no fire → stream continues | ✅ |
| 4 | **Email verify ενώ chatsProvider active** | `userChanges()` emits updated user → listener fires `emailVerifiedChanged=true` → invalidate → νέο `streamChats()` περνά `canUserCommunicate` | ✅ |
| 5 | **Widget tree rebuild** ενώ listener τρέχει | `mounted` guard protects setState | ✅ |
| 6 | **unreadBadgeProvider re-evaluation** | `chatsProvider` invalidated → unreadBadgeProvider dirty → re-evaluates once → correct badge | ✅ |
| 7 | **ChatListScreen open during sign-out** | `chatsProvider` invalidated → ChatListScreen rebuilds empty → GoRouter redirect | ✅ |
| 8 | **Network loss during sign-out** | `userChanges()` emits null from local SDK state → immediate | ✅ |
| 9 | **Sign-in with already-verified email** | `uidChanged=true + emailVerifiedChanged=true` → ΜΙΑ invalidation | ✅ |
| 10 | **Hot reload** | Listener re-runs → fires with prev=undefined, next=currentUser → invalidate. Same as today (ref.watch also caused this) | ✅ |
| 11 | **Process kill/restart** | Fresh start → fresh listener → fresh chatsProvider | ✅ |
| 12 | **Sign-out from settings_screen while signOut is in-flight** | `isSigningOut` guard + listener handles the null emission | ✅ |
